## 目前最可信的策略結論

1. **5,000 張以上應作為主池**  
   walk-forward 驗證後，唯一可安全升級的是 5,000 張以上主池。2,000～4,999 張只作備選或觀察。

2. **D1 blowoff 後 D2+ 重返警示價有訊號，但不是高勝率策略**  
   P1.5 保守回測顯示成本後勝率約 40.7%，平均報酬為正但中位數為負。

3. **P1.6 的小樣本好規則不可直接升級**  
   例如 `5,000張 + gap 0～3% + 量比 >3x` 驗證期好看，但探索期樣本太少。

4. **A 型策略目前沒有被充分測到**  
   V1 selected 候選幾乎都是 B 型，A 型樣本太少，進 P2 後要獨立修正。

5. **大盤 regime 值得做，但日線 proxy 不夠**  
   P1.8 顯示 `TREND_UP` 比 `NEUTRAL` 好，但仍不足以讓中位數轉正。P2 必須使用 09:15 分時大盤。

6. **EPS／處置風險欄位已接線，但歷史 as-of 不完整**  
   可供每日 production 使用，但不能拿 current snapshot 回推歷史。

## P2 入口條件

P2 可以開始，因為 P1 已回答兩件事：

- 策略有可觀察訊號，但需要更嚴格過濾。
- 日線資料不足以判斷真實當沖，必須進入每日系統與分時驗證。

## P2 建議順序

1. **P2.0 建立每日候選股 brief prototype**
   - D0 盤後候選。
   - D1 觀察清單。
   - D2 重返警示價清單。
   - 產生 Markdown/CSV 作為 dashboard 與通知的上游資料。

2. **P2.1 每日晚間資料更新 pipeline**
   - 盤後更新當天 TWSE/TPEx 行情。
   - 重建事件表、D1/D2 觀察與 data health。
   - 加入異常跳空/除權息檢查，避免把價格基準變動誤判成交易訊號。

3. **P2.2 Dashboard MVP**
   - 在 GitHub Pages 上顯示每日候選、D1 觀察、D2+ 觀察。
   - 顯示處置風險、EPS 虧損、警示價、停損價、失效價與下一步建議。
   - 顯示資料健康狀態，包含資料日、缺漏市場、是否 trade-ready。

4. **P2.3 接 09:15 大盤 regime**
   - 強盤／弱盤／大跌。
   - 驗證原始強弱盤量縮/放量假設。

5. **P2.4 補分時資料**
   - 解決進場價與停損價同日觸發的先後順序。
   - 建立真實當沖回測。

6. **P2.5 每日風險封存**
   - 注意/處置快照每日封存。
   - EPS as-of 資料擴充。

7. **P2.6 通知系統**
   - 晚間候選通知。
   - D1 盤中/盤後更新。
   - D2 觀察提醒。

8. **P2.7 Dashboard production 化**
   - 日期切換與最近 20 個交易日回看。
   - 個股狀態歷程：D0 候選 → D1 觀察 → D2+ 觸發/失效。
   - 排序、篩選、匯出與 UI smoke test。

9. **P2.8 paper trading**
   - 固定規則 forward test 20～60 個交易日。
   - 不在測試中途調參。

## 不建議現在做的事

- 不建議直接用 P1.5 結果進實戰。
- 不建議把小樣本組合升級成正式交易規則。
- 不建議未接分時資料就宣稱當沖勝率。
- 不建議用 current warning snapshot 回推歷史處置狀態。
