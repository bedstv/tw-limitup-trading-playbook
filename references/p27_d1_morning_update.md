# P2.7 D1 09:15 更新

## 日期對齊

`candidate_date` 是 D0 盤後候選日，`decision_date` 必須是其後的實際交易日。
例如 2026-07-09 盤後候選只能使用 2026-07-10 的 09:15 資料。

## 自動程序

macOS LaunchAgent `com.bedstv.twlimitup.d1-0915` 於週一至週五 09:16 執行：

1. 從 Dashboard 索引找出最新的前一候選日。
2. 取得當日 09:15 TAIEX regime。
3. 在 09:15–09:17 取得候選股即時快照。
4. 套用公司行動、處置、跳空與大盤 routing。
5. 更新原候選日 JSON，提交並推送 GitHub Pages。

個股 MIS 不提供可依任意日期回補的 09:15 序列，因此錯過擷取時間後程序會失敗，
不會使用盤中較晚價格冒充 09:15。

## D1 狀態

- `WATCH`：可進入後續觸價與量價確認。
- `PULLBACK_ONLY`：開盤跳空超過 5%，禁止直接追價。
- `DOWNRANK`：市場型態與候選 setup 不匹配。
- `REJECT`：跳空超過 7%、公司行動、處置風險或弱勢盤必要條件不成立。
