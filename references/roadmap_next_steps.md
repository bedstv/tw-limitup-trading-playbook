# 後續工作清單

此清單記錄系統從 P2.0 原型推進至可用交易研究系統的過程。

> **2026-07-15 開發基準：**「正式後續開發順序」是後續工作的唯一執行依據。
> 下方 P2.1～P2.11 保留歷史決策與實作紀錄；若舊的「待做」內容與新基準衝突，
> 以正式後續開發順序為準。使用者後續要求「進行下一步」時，依尚未完成的最前項繼續。

## 正式後續開發順序

### 已完成基礎

- P1：全市場日線資料、事件回測、保守回測與 walk-forward 驗證。
- P2.1～P2.10：每日資料、風險封存、公司行動、09:15 判斷、Dashboard 與 Telegram。
- P2.11：固定規則 `p2.11_v2` 紙上交易框架已建立，目前持續累積樣本。
- P2.12～P2.19：回測摘要、紙上交易驗證頁、更新履歷、板塊共識、載入失敗保護、
  可分享篩選網址、Telegram 看盤連結、個股報價備援與排程失敗 Telegram 告警。

### P2.20 自動化可靠性強化（目前最高優先）

目的：每天能準時、可追溯地完成「資料產生 → GitHub 發布 → Telegram 發送 → Dashboard 可讀」。

開發項目：

1. **P2.20.1 排程執行版本統一與預檢**（已完成）
   - 保留 ASCII runtime，避免 `launchd` 直接使用中文路徑。
   - 每次執行前記錄資料程式、Pages 與 wrapper 的 Git commit／版本。
   - 開盤前與盤後前檢查 LaunchAgent 是否載入、路徑是否存在、版本是否落後。
2. **P2.20.2 自動重試與通知去重**（已完成）
   - D1 09:16 失敗後，在有效資料時間內自動重試一次。
   - 盤後流程失敗後自動重試一次。
   - 同一階段、同一交易日只送一份交易摘要；失敗告警與恢復通知分開記錄。
3. **P2.20.3 端到端健康檢查**（已完成）
   - 驗證每日資料、遠端 Git commit、Dashboard 可讀性與 Telegram delivery manifest。
   - 健康狀態顯示排程時間、實際完成時間、版本與失敗步驟。

完成標準：連續至少 10 個交易日不需人工回補；失敗時 5 分鐘內收到 Telegram 告警。

### P2.21 交易日曆與資料新鮮度（已完成）

- 接入例假日、颱風停市、臨時休市與補班交易日判斷。
- 明確判斷目前「應有 09:15 資料」「應有盤後資料」或「正常休市」。
- Dashboard 顯示預期資料日、最新資料日與是否過期。

完成標準：休市不誤報、交易日缺資料能告警，且不產生假的當日 Dashboard。

### P2.22 D1 個股分時資料覆蓋（已完成）

- 固定 `TWSE MIS → Fugle 1 分鐘 K → 排除單檔` 的來源順序。
- 每檔封存實際資料來源、時間、有效 K 棒數與失敗原因。
- 顯示每日候選覆蓋率；任何單檔失敗不得中斷整批候選與 Telegram。

完成標準：有資料才判斷；缺資料維持排除，不以推估值或日線替代 09:15 訊號。

### P2.23 紙上交易正式累積（已完成）

- 固定 `p2.11_v2`，測試期間不得調參。
- 每日記錄候選、09:15 決策、觸發、停損、留倉覆核、結算與資料不足。
- 先完成 20 個 D1 判斷日中期檢查，目標累積 60 日後正式評估。
- 第 20 個 D1 判斷日完成時，盤後 Telegram 會提示進行中期人工檢查；第 60 日會提示正式評估，兩者皆不會自動調參、升級或下單。

完成標準：所有未成交、資料不足、未結算紀錄皆保留，不得視為 0% 報酬。

### P2.24 策略成效評估頁（已完成）

- 顯示平均與中位數淨報酬、勝率、停損率、期望值與最大回撤。
- 依 A/B 型、大盤強弱、5,000 張主池、板塊共識與 D1 決策拆分。
- 與 P1 日線回測比較，說明 forward test 是否符合預期。
- 20 日為中期人工檢查；60 日且未結算紀錄完成覆核後，才顯示正式人工評估門檻。系統不會自行通過策略或啟動下單。
- Dashboard 提供人工評估清單，逐項顯示樣本日數、待結算紀錄與已結算交易數，避免只憑單一報酬數字決策。

完成標準：樣本不足時只顯示「收集中」；達門檻後才產出保留、研究或淘汰建議。

### P2.25 歷史分鐘線回測（覆蓋審計完成；歷史回放仍受資料限制）

- 評估可合法長期保存的歷史 1 分／5 分 K 資料源與費用。
- 建立 coverage manifest 後，重跑 09:16 後進場、停損與留倉順序。
- 若沒有足夠歷史分鐘資料，維持 forward paper trading，不宣稱完成當沖回測。

### P2.26 A 型策略研究（已完成；維持研究用途）

- 維持獨立研究候選池，不影響 production V1/V2。
- 重新驗證量縮盤整定義與樣本數。
- 驗證期至少 10 筆實際成交前不得升級上線。

### P2.27 Dashboard 決策簡化（已完成）

- 增加今日重點摘要、逐檔資料來源、異常原因與交易限制。
- 保持電腦表格與手機卡片兩種呈現。
- 目標是在手機一頁內回答「今天看什麼、為什麼、停損在哪」。
