# P1.7 walk-forward 驗證

目的：檢查 P1.6 子集合規則是否只是樣本內剛好好看。這一步固定候選規則，不再用驗證期挑新規則。

## 切分方式

- 探索期：2016～2021
- 驗證期：2022～2026
- 最低驗證期進場數：10 筆

## 執行指令

共用資料層：`../tw-market-data`

```bash
python3 scripts/validate_p17_walk_forward.py \
  --trades-tag p15_2016_2026_selected \
  --output-tag p17_2016_2026_selected \
  --explore-start-year 2016 \
  --explore-end-year 2021 \
  --validate-start-year 2022 \
  --validate-end-year 2026 \
  --min-validate-executed 10
```

## 產出

- walk-forward 統計：`../tw-market-data/data/processed/walk_forward_stats_p17_2016_2026_selected.csv`
- 規則決策表：`../tw-market-data/data/processed/walk_forward_decisions_p17_2016_2026_selected.csv`
- 健康報告：`../tw-market-data/data/manifests/p17_2016_2026_selected_walk_forward_health.json`

|---|---:|---:|---:|---:|---:|---:|

## 規則驗證結果

## 重要結論

1. **可升級的只有 5,000 張以上主池**  
   驗證期進場 107 筆，樣本數足夠；相較 baseline，中位數略改善、平均報酬略改善、停損率略低。

2. **P1.6 看起來好的量比 2～3x，在驗證期失效**  
   探索期中位數為 +1.12%，但驗證期中位數降為 -2.91%，比 baseline 更差。這是 walk-forward 抓出樣本內幻覺的典型例子。

3. **小樣本組合不能升級為正式規則**  
   例如 `5,000張以上 + gap 0～3% + 量比 >3x` 驗證期表現好，但探索期只有 1 筆進場。這只能列入觀察，不可視為穩健規則。

4. **目前策略仍非高勝率型**  
   即使只看驗證期，baseline 與主池中位數仍為負。策略價值若存在，可能來自少數右尾大漲事件，而不是穩定小賺。

## V1.1 建議調整

基於 P1.7，可安全採用的改動只有：

- 候選主池改為 **日成交量 5,000 張以上**。
- 2,000～4,999 張僅作備選觀察，不進主要回測池。
- D1 blowoff 後的交易邏輯仍需搭配下一層 filter，不宜直接交易。

暫不升級：

- 量比 2～3x 作為正式交易條件。
- gap 0～3% + >3x 等小樣本組合。
